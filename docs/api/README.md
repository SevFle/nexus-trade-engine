# API reference

The Nexus Trade Engine exposes a FastAPI app at `engine.app:create_app`,
served by uvicorn. Every route is mounted under the
[`api_router`](../../engine/api/router.py); the bulk of the surface
lives under `/api/v1/...`. The OpenAPI schema is auto-generated by
FastAPI and served at `/openapi.json` (and `/docs` for the Swagger UI)
in non-production environments.

This directory documents each route group in one Markdown file. Every
file is structured the same way:

1. **Routes** — a table of method → path → auth requirement → summary.
2. **Request/response shapes** — Pydantic models, copied verbatim from
   `engine/api/routes/<area>.py` and trimmed to what callers actually
   need to know.
3. **Examples** — `curl` calls that work against a fresh `make dev`
   instance.
4. **Errors** — non-2xx codes the route can return and the conditions
   that trigger them.

## Common conventions

### Authentication

Every protected route accepts one of two credentials in the
`Authorization` header (or `X-API-Key`):

| Credential                | Header                                       | Issued by                       | Notes |
|---------------------------|----------------------------------------------|---------------------------------|-------|
| **JWT access token**      | `Authorization: Bearer <jwt>`                | `POST /api/v1/auth/login` or `/refresh` | Short-lived (default 1 h). Issued to interactive users (browser, MCP). |
| **Engine API key**        | `Authorization: Bearer nxs_<env>_<random>` or `X-API-Key: nxs_...` | `POST /api/v1/auth/api-keys` | Long-lived, scoped. The full token is shown once at creation. |

Auth resolution lives in
[`engine/api/auth/dependency.py:get_current_user`](../../engine/api/auth/dependency.py).
Token shape dispatches on the `nxs_` prefix; everything else is treated
as a JWT.

### Roles

Roles are a *hierarchy* (not an equality check). The hierarchy in
`engine/api/auth/dependency.py` is:

```
viewer (0) → user (1) → retail_trader (2) → quant_dev (3) → developer (4)
          → portfolio_manager (5) → admin (6)
```

A `Depends(require_role("quant_dev"))` accepts users with `quant_dev`,
`developer`, `portfolio_manager`, or `admin`. Anything below fails with
`403 Forbidden`.

### Scopes (API keys only)

API keys declare a scope list. JWT-authenticated requests bypass scope
checks (the role check is the gate); API-key requests are checked
against:

| Scope  | Allows                                            |
|--------|---------------------------------------------------|
| `read` | `GET` / `HEAD` only.                              |
| `trade`| `read` + write paths (backtest, portfolio, webhooks). |
| `admin`| Equivalent to the `admin` role; supersedes both.  |

Use `Depends(require_api_scope("trade"))` for write paths that may be
called by automation. See `engine/api/auth/dependency.py`.

### Legal acceptance gate

Some routes are mounted with
`dependencies=[Depends(require_legal_acceptance)]` (see
[`engine/api/router.py`](../../engine/api/router.py)). Those routes
return `403 legal_acceptance_required` until the calling user has
accepted the current version of every document flagged
`requires_acceptance=true`. Acceptance is recorded via
`POST /api/v1/legal/accept`.

Gated routes today: `backtest`, `scoring`, `market-data`, `portfolio`,
`strategies`, `marketplace`, `webhooks` (read OK without acceptance;
write requires it).

### Error envelope

There is no formal error envelope. Routes return either:

- A Pydantic model on success (2xx).
- A FastAPI default `{"detail": "..."}` JSON body on 4xx/5xx.
- A `JSONResponse` with `status`, `error`, and route-specific fields on
  the backtest result endpoint (intentional — the shape must round-trip
  through the React dashboard's typed fetcher).

Validation errors from Pydantic come back as
`{"detail": [{"loc": [...], "msg": "...", "type": "..."}, ...]}` per
[RFC 7807](https://datatracker.ietf.org/doc/html/rfc7807) (the shape
FastAPI emits by default).

### Rate limiting

A token-bucket limiter runs in front of every route. Defaults:

- 600 requests / minute, burst 60.
- Exempt: `/health`, `/health/providers`, `/ready`, `/metrics`.
- Per-route overrides today: `/api/v1/client/errors` is capped at
  30 req/min/IP so a buggy React render loop cannot DoS the log
  pipeline.

429 responses include `Retry-After`. See
[`engine/api/rate_limit.py`](../../engine/api/rate_limit.py) for the
full override table.

### Body size

A 1 MiB hard cap is enforced by `BodySizeLimitMiddleware` on every
request. Routes that legitimately need more (CSV upload, large
backtest configs) will need a per-route override before they ship.

### Request id / correlation id

The `CorrelationIdMiddleware` reads `X-Request-ID` from the inbound
request (or generates one) and echoes it back on every response. Logs
include the same id in structlog's `request_id` field. Pass it through
in client code if you want to correlate distributed traces.

### Content type

Request bodies are JSON (`application/json`). Multipart / form-data is
not currently used. CSV downloads come back with `Content-Type: text/csv`
and a `Content-Disposition: attachment` header — see `tax.md` for the
canonical example.

## OpenAPI vs. this directory

This directory is the *human* documentation. The OpenAPI schema is the
*machine* documentation. They should agree but they will drift — when
you change a route, change both. The reviewer checklist for any
route-touching PR should include "did you update `docs/api/<area>.md`?"
